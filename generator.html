GTS Data Generator

Build gts-mods.json

Drop the collection owner's vortex.deployment.json in here. It reads every properly-named mod and builds the fingerprint database the Decoder uses — so GTS users get exact names for the whole collection. Run this once per collection revision.

  1. 1Get the owner's vortex.deployment.json (their file has clean names, not codes).
  2. 2Set the revision number, then drop the file below.
  3. 3Copy or download the generated gts-mods.json and commit it to the repo.

How it works: for every mod in the owner's file it extracts a clean name (stripping the Nexus ID and version), the Nexus mod link where present, and a fingerprint (the PBRNifPatcher json name, a distinctive PBR folder, or a plugin). Only PBR/PGPatcher-relevant mods are included — the ones that can show a code for other users.
